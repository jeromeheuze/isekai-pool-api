CPU mining setup

What is CPU mining?

Yespower and yescrypt families are built so ordinary CPUs stay competitive — GPUs and ASICs don’t dominate the way they do on SHA-256 or Ethash.

You run a miner on hardware you already own: a Raspberry Pi, an old laptop, or a desktop. Your machine talks to a coin node over RPC (or a pool) and hashes blocks.

isekai-pool.com hosts public RPC nodes so you can solo-mine or build tools without syncing a full node yourself. See the status dashboard and API docs.

Install cpuminer-opt

sudo apt update && sudo apt install -y git build-essential automake autoconf \
  libcurl4-openssl-dev libssl-dev libjansson-dev libgmp-dev zlib1g-dev screen

git clone https://github.com/JayDDee/cpuminer-opt.git
cd cpuminer-opt
./autogen.sh
./configure CFLAGS="-O3 -march=armv8.2-a+crypto -mtune=cortex-a76"
make -j4

Use pre-built releases from the cpuminer-opt project when available. Linux is recommended for long-running nodes.

Run the miner

Use your coin’s algorithm, RPC host/port, and credentials. Host 153.75.225.100 — ports: 9982 (YTN), 8432 (KOTO), 9368 (TDC). More on Node Status.

./cpuminer \
  --algo=ALGO \
  --url=http://153.75.225.100:9982 \
  --user=YOUR_RPC_USER \
  --pass=YOUR_RPC_PASS \
  --coinbase-addr=YOUR_ADDRESS \
  --threads=4

Tips

  • Use screen or tmux so mining survives SSH disconnect.
  • --cpu-priority=2 can keep the system responsive.
  • Watch for accepted in logs when you find shares/blocks.
  • Raspberry Pi: vcgencmd measure_temp · Linux: sensors
